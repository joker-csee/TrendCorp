## 二、五大子系统

### 子系统 A：数据中心

**职责**：汇聚所有原始数据，输出标准化数据接口。

| 数据模块 | 数据项 | 更新频率 | 数据源（示例） |
|---------|--------|---------|-------------|
| 个股行情 | 日K（OHLCV）、分时数据、均线值（MA5/10/21/55） | 日终/盘中 | 东方财富 API / Tushare |
| 板块行情 | 申万行业指数、概念板块指数、板块内成分股列表 | 日终 | 同花顺行业 / 申万指数 |
| 财务数据 | 营收增速、净利润增速、ROE、毛利率 | 季报后更新 | 东方财富财务数据 |
| 资金流 | 板块主力净流入、个股大单净流入/流出 | 日终 | 同花顺资金流 |
| 成交量 | 个股近 5/20 日均量、板块成交额、全市场成交额 | 日终 | 行情数据计算 |
| 公告事件 | 业绩预告、重大合同、分析师覆盖 | 实时 | 巨潮资讯 / 东方财富公告 |

### 子系统 B：主线识别引擎

**职责**：从全市场 ~200 个行业中，识别出 1-3 条主线板块。

#### B-1 板块扫描器

```
输入：全市场申万行业 + 概念板块（~200 个）
输出：候选主线列表（≤10 个）

扫描指标（每周执行一次）：

指标 1：趋势强度
  - 板块指数 MA5 > MA10 > MA21 > MA55 → 全部满足得 4 分
  - 每缺一级扣 1 分
  - 阈值：≥ 3 分进入候选

指标 2：相对强度
  - 近 20 日板块涨幅 / 同期沪深 300 涨幅
  - 阈值：比值 > 1.5 进入候选

指标 3：资金确认
  - 近 5 日板块主力资金净流入 > 0
  - 近 20 日板块成交额 / 全市场成交额 ≥ 板块流通市值 / 全市场流通市值
  - 阈值：两项都满足进入候选

指标 4：梯队完整性
  - 板块内近 5 日出现过涨停（≥ 10cm）的个股数 ≥ 3
  - 板块内存在流通市值 > 100 亿的标的
  - 阈值：两项都满足进入候选

评分 = 趋势强度 × 0.35 + 相对强度 × 0.25 + 资金确认 × 0.25 + 梯队完整性 × 0.15
候选阈值：总分 ≥ 0.6 进入候选主线
```

#### B-2 主线确认器

```
输入：B-1 的候选主线列表
输出：确认主线（≤3 个）+ 观察主线（≤5 个）

在 B-1 评分基础上，叠加"持续性验证"：

验证项 1：已连续跑赢大盘 ≥ 3 周 → +1 分
验证项 2：有可辨识的产业逻辑（非纯题材）→ +1 分
验证项 3：板块成交额未出现"天量"（近 5 日均量 / 近 20 日均量 < 2.5）→ +1 分
验证项 4：板块 PE 分位 < 90%（非极端拥挤）→ +1 分

确认主线：总分 ≥ 3 分
观察主线：总分 2-2.99 分
排除：总分 < 2 分
```

#### B-3 主线退潮预警器

```
输入：当前持有的确认主线
输出：退出/减仓/继续持有信号

预警信号（满足 ≥ 2 项 → 减仓；满足 ≥ 3 项 → 退出）：

🔴 板块指数连续 3 天跑输沪深 300
🔴 板块近 5 日涨停家数 < 2（"先锋"灭绝）
🔴 板块成交额较 20 日均量萎缩 > 40%（人气退散）
🔴 中军标的出现"放量长阴"（跌幅 > 5% + 成交量 > 20 日均量的 2 倍）
🟡 板块 PE 分位 > 95%（极端拥挤）
```

### 子系统 C：中军筛选引擎

**职责**：在确认主线的板块内，锁定 1-2 只中军标的。

#### C-1 中军评分模型

```
输入：确认主线板块的所有成分股
输出：中军评分排序（每个板块只取前 2 名）

评分维度（每项 0-1 分，加权平均）：

维度 1：市值规模（权重 0.15）
  流通市值 100-500 亿 → 1.0
  流通市值 50-100 亿或 500-1000 亿 → 0.6
  流通市值 < 50 亿或 > 1000 亿 → 0.2

维度 2：流动性（权重 0.20）
  近 20 日均成交额在板块内排名前 5 → 1.0
  排名 6-10 → 0.5
  排名 > 10 → 0.1

维度 3：均线结构（权重 0.25）
  MA5 > MA10 > MA21 > MA55 全部满足 → 1.0
  仅 MA5、MA10、MA21 多头 → 0.6
  MA21 > MA55 但短期缠绕 → 0.3
  均线空头 → 0

维度 4：量价健康度（权重 0.25）
  上涨日平均成交量 / 回调日平均成交量 > 1.5 → +0.5
  最近一次回调缩量至前期高峰的 50% 以下 → +0.5

维度 5：基本面质量（权重 0.15）
  最近季度归母净利润同比 > 0 → 0.5
  最近季度营收同比 > 0 → 0.5

中军评分 ≥ 0.65 → 合格中军
```

#### C-2 均线状态监控器

```
输入：已选中的中军标的
输出：均线结构和买卖点信号

均线结构每日体检：
  MA5/MA10/MA21/MA55 数值 + 斜率方向

买卖点信号：
  🟢 买点一（最佳）：
      价格距 MA10 偏差 < 3%
      + 当日成交量 < 20 日均量的 60%
      + K 线收小阳/十字星
      → 信号：A级买点
  
  🟢 买点二（可用）：
      板块内 3 只以上同步放量
      + 中军涨幅 3-5%
      + 成交量 > 5 日均量的 1.5 倍
      → 信号：B级买点（仓位减半）
  
  🟡 持有：
      收盘价在 MA10 上方
      + 无退潮预警
      → 信号：持有

  🔴 减仓：
      收盘价跌破 MA10 → 减半仓
      收盘价跌破 MA21 → 清仓
```

### 子系统 D：买卖执行引擎

**职责**：根据信号生成交易指令，管理仓位和订单。

#### D-1 仓位管理器

```
输入：当前净值、当前持仓、新信号
输出：目标仓位和交易指令

仓位上限规则：
  总仓位上限：    70%（永不满仓）
  单板块上限：    35%
  单票上限：      30%
  现金最低保留：  30%

建仓规则：
  首次建仓：目标仓位的 50%
  确认加仓：回踩 MA10 企稳后，加至目标仓位的 100%
  追入建仓（B级买点）：目标仓位的 30%（再确认后加至 60%）

减仓规则：
  MA10 破位 → 减当前持仓的 50%
  MA21 破位 → 剩余全部清仓
  板块退潮预警 ≥ 2 项 → 减该板块全部持仓的 50%
  板块退潮预警 ≥ 3 项 → 清仓该板块
```

#### D-2 订单执行器

```
交易指令格式：
  {标的代码} | {方向} | {目标仓位%} | {价格类型} | {有效期}

价格类型规则：
  低吸（A级买点）：限价单，价格为 MA10 上浮 0.5%
  追入（B级买点）：限价单，价格为当前价
  止损：         市价单（不设限价，保命优先）

执行时段：
  建仓/加仓：   盘中 10:00-14:30（避开早盘量化和收盘博弈）
  止损：       任何时候触发即执行
  止盈减仓：   尾盘 14:50 前执行

⚠️ 禁止：
  早盘 9:30-9:45 下单
  收盘 14:55 后追单
  市价追涨（永远用限价单）
```

### 子系统 E：风控监控引擎

**职责**：全时段监控风险，触发熔断时强制执行。

#### E-1 三级熔断器

```
═══════════════════════════════════════
第一级：个股止损（自动触发，即时执行）
═══════════════════════════════════════
触发条件（任一满足即触发）：
  □ 收盘价跌破 MA10 → 次日开盘减半仓
  □ 收盘价跌破 MA21 → 次日开盘全清
  □ 单票浮亏 ≥ 8% → 即时市价清仓（不等收盘）
  □ 标的出现利空公告 → 次日竞价出清

═══════════════════════════════════════
第二级：日/周熔断（自动触发，限制开仓）
═══════════════════════════════════════
触发条件：
  □ 当日总净值回撤 ≥ 3% → 次日禁止开新仓
  □ 连续 2 笔止损 → 次日禁止开新仓
  □ 当周净值回撤 ≥ 5% → 下周禁止开新仓

═══════════════════════════════════════
第三级：月/策略熔断（手动触发，强制空仓）
═══════════════════════════════════════
触发条件：
  □ 当月回撤 ≥ 10% → 当月停止交易，下月恢复
  □ 策略总回撤 ≥ 15% → 暂停，需人工复核后再决定是否继续

解禁条件（月/策略熔断）：
  □ 空仓至少 5 个交易日
  □ 完成复盘报告：回撤原因、策略问题、改进方案
  □ 至少等待一个"非冲动"状态确认（如休息一个周末）
```

## 四、周度 / 月度例行操作清单

### 每周日（30 分钟）

```
□ 运行 B-1 板块扫描器 → 更新候选主线
□ 运行 B-2 主线确认器 → 确认本周主线
□ 检查现有持仓：标的还在确认主线板块内吗？
    - 不在 → 下周找机会退出
    - 在 → 继续持有
□ 运行 C-1 中军评分 → 确认本周中军标的池
□ 检查下周财报/事件日历
```

### 每个交易日（10 分钟）

```
□ 开盘前：过一遍 E-2 监控仪表盘
□ 盘中：只盯 C-2 的买卖点信号，其他时间不看盘
□ 收盘后：
    - 更新 C-2 均线状态（所有持有标的 + 候选池标的）
    - 运行 B-3 退潮预警
    - 如果交易了 → 写 F-1 日志（5 分钟）
```

### 每月末（1 小时）

```
□ 运行 F-2 月度绩效归因
□ 统计胜率、盈亏比、最大回撤
□ 统计违规操作次数（这是最重要的指标）
□ 写一句"下月最需要改进的一点"
```

## 五、关键设计原则

1. **机械执行 > 主观判断**：信号触发就做，没触发就不做。"我觉得"是最大敌人。
2. **先守后攻**：风控（子系统 E）权限高于一切其他子系统。熔断触发时，买卖引擎必须服从。
3. **低频 > 高频**：这个系统每周只需要 30 分钟 + 每天 10 分钟。如果发现自己每天在看盘超过 1 小时，说明在执行偏离。
