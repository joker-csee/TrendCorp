## 二、策略原理（5 分钟理解）

### 2.1 核心理念

A 股每年都有 1-3 条**主线**——由产业趋势驱动的、持续数月甚至数年的板块行情（例如 2025-2026 年的 AI 算力）。本系统帮你做三件事：

1. **识别主线**：从 200+ 个板块中自动找出当前最强的主线
2. **锁定中军**：在主线中找到市值适中、流动性好、机构重仓的核心标的
3. **执行买卖**：根据均线信号告诉你什么时候买、什么时候卖

### 2.2 关键概念

| 概念 | 通俗解释 |
|------|---------|
| **主线** | 当前资金最集中、趋势最强的 1-3 个板块 |
| **中军** | 主线中市值 100-500 亿、成交量前 5 的核心标的 |
| **A 级买点** | 股价回踩 10 日均线 + 缩量企稳 = 低吸机会 |
| **B 级买点** | 板块内 3 只以上同步放量启动 = 趋势确认 |
| **MA10 破位** | 收盘价跌破 10 日线 = 减半仓 |
| **MA21 破位** | 收盘价跌破 21 日线 = 全部清仓 |

### 2.3 风控规则（必须遵守）

| 规则 | 说明 |
|------|------|
| 总仓位 ≤ 70% | 永远保留 30% 现金 |
| 单票 ≤ 30% | 不重仓单只股票 |
| 单板块 ≤ 35% | 不押注单一板块 |
| 个股 -8% 止损 | 亏损达 8% 无条件清仓 |
| 日回撤 -3% | 当天禁止开新仓 |
| 月回撤 -10% | 当月强制空仓 |

## 三、每日操作流程

### 3.1 上午（5 分钟）

1. 打开浏览器访问 `http://localhost:8000`
2. 查看**仪表盘**首页的净值卡片和风控状态
3. 如果风控显示 "DAILY_BAN" 或红色 — **今天不交易**

### 3.2 盘中（按需查看）

1. 打开**扫描页**（`/scanner`），查看当前确认主线
2. 如果主线列表为空或显示"数据不可用"— 点击"手动触发扫描"按钮
3. 打开**持仓页**（`/positions`），查看当前持仓的信号变化：
   - 出现 **REDUCE** 信号 → 在券商 App 卖出一半
   - 出现 **EXIT** 信号 → 在券商 App 全部清仓

### 3.3 收盘后（5 分钟）

1. 回到仪表盘首页，查看今日收益
2. 如果当天有交易：
   - 打开**日志页**（`/journal`）
   - 点击"+ 录入新交易"，填写开仓/平仓信息
3. 查看**净值曲线**是否正常更新

## 四、页面说明

### 4.1 仪表盘（`/`）

- **净值卡片**：总净值、今日/本周/本月收益、仓位、可用现金
- **风控状态**：绿色=正常 / 橙色=预警 / 红色=熔断
- **净值曲线**：近 90 天净值走势图
- **持仓表**：当前持仓的代码、浮盈、MA10 状态
- **信号表**：当前主线的中军买入信号

### 4.2 主线扫描（`/scanner`）

- 点击"手动触发扫描"执行一次全市场板块扫描
- 表格显示板块评分（趋势/相对/资金/梯队）和确认状态
- 确认主线 → 进入中军筛选；观察主线 → 保留关注；排除 → 不做

### 4.3 持仓（`/positions`）

- 当前持仓列表（ID、成本、占比、浮盈、MA10 状态）
- 风控事件历史（时间、类型、级别、详情）
- 页面每 60 秒自动刷新

### 4.4 交易日志（`/journal`）

- **录入交易**：填写股票代码、开仓价格、开仓理由、MA10 值、仓位占比
- 交易记录会生成唯一编号（格式 T-YYYYMMDD-NNN）
- 平仓通过 API 更新（`POST /api/trade/close`），后续版本将增加 Web 表单

### 4.5 月度报告（`/report`）

- 月度交易统计（笔数、胜率、月收益、盈亏比）
- 最大单笔盈利/亏损
- 违规次数统计
- 按买点类型归因（A 级/B 级对比）

## 五、常见问题

### Q: 仪表盘显示"数据不可用"怎么办？

A: 通常是 akshare 网络问题。点击 Scanner 页面的"手动触发扫描"重试。如果持续失败，检查网络环境是否能访问 eastmoney.com。

### Q: 信号显示 A_BUY，但我不想买怎么办？

A: 系统提供的是**建议**，不是自动执行。最终买卖决策由你做出。系统只帮你缩小候选范围、给出买卖参考。

### Q: 亏损达到 8% 但均线还没破位，要不要卖？

A: 系统规则是**亏损 -8% 优先于均线信号**（BRT-06）。达到 -8% 无条件清仓。

### Q: 为什么有些板块突然从"确认主线"变成"观察"或"排除"？

A: 主线确认是每周更新的。可能触发了退潮预警（连续跑输大盘、涨停家数不足、成交额萎缩等）。

### Q: 系统能自动帮我下单吗？

A: v1.0 不支持。系统生成交易指令（股票代码、方向、仓位、限价），你在券商 App 手动执行。这是有意为之——决策权归人。

## 六、配置文件

### 6.1 .env 环境变量

```bash
DATABASE_PATH=data/trend_corp.db   # 数据库路径
LOG_DIR=logs                        # 日志目录
LOG_LEVEL=INFO                      # 日志级别 (DEBUG/INFO/WARNING/ERROR)
INITIAL_CAPITAL=100000              # 初始本金
```

### 6.2 config.py 策略参数

```python
# 板块扫描权重
trend: 0.35      # 趋势强度
rel_strength: 0.25  # 相对强度
fund: 0.25       # 资金确认
echelon: 0.15    # 梯队完整性

# 仓位约束
total_cap: 0.70    # 总仓位上限
single_cap: 0.30   # 单票上限
sector_cap: 0.35   # 单板块上限
cash_min: 0.30     # 最低现金保留

# 风控
stock_loss_limit: -0.08   # 个股止损线
daily_dd_limit: -0.03     # 日熔断线
monthly_dd_limit: -0.10   # 月熔断线
```

修改参数需在 `config.py` 或 `.env` 中更改后重启服务。

## 八、注意事项

1. **首次使用**务必先执行 `python main.py` 初始化数据库
2. **板块数据**依赖网络，首次拉取约需 30-60 秒
3. **不要在生产环境暴露 8000 端口**到公网——这是本地工具
4. **参数修改**建议按季度评估，不要每次亏损后就调参数
5. **净值追踪**依赖你手动录入交易（Journal 页面）——如果忘记录入，净值会偏离实际
